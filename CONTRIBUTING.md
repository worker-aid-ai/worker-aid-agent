# 贡献指南

感谢参与 Worker Aid Agent。本项目面向普通劳动者、公益法律服务和开源协作场景，所有贡献都应优先保护劳动者隐私，并帮助使用者获得更清楚、可复核的材料，而不是替代专业法律判断。

## 可以贡献什么

- 场景数据：新增常见劳动争议类型、分流提示和风险提醒。
- 证据清单：补充某类争议通常需要整理的材料。
- 模板：优化投诉、仲裁、法律援助咨询摘要、证据目录等草稿。
- 数据源：补充最低工资、法律援助、劳动监察、人社服务、仲裁入口等公开信息索引。
- 文档：补充本地化办事指南、使用教程、开发说明和安全提示。
- 代码：改进 CLI、金额计算器、校验器、Web UI、导出器、数据更新脚本。

## 开发环境

要求：

- Node.js 18+
- pnpm 11+

首次安装：

```bash
pnpm install
```

常用命令：

```bash
pnpm run validate
pnpm test
pnpm run demo
pnpm run serve:web
```

本地 Web 界面默认访问：

```text
http://localhost:5173/
```

## 代码贡献要求

- 保持核心 CLI 零运行依赖，除非有很强理由。
- CLI 命令、JSON 输入、Markdown/HTML 导出应尽量保持结构化、可复核。
- 修改金额计算、最低工资查询、导出器或校验逻辑时，应补充或更新 `tests/*.test.mjs`。
- 不要引入会主动上传用户案情、证据或敏感信息的默认行为。
- 新增功能如果会影响法律输出，必须保留不确定性、核验提醒和人工复核提醒。

## 内容与法律信息要求

- 不提交真实个人信息、未脱敏案例、身份证号、手机号、住址、银行卡号、公司内部敏感文件原文。
- 不提交规避法律、伪造证据、隐匿事实、骚扰施压、承诺结果或冒充法律专业人士的内容。
- 法律依据、政策口径和办事入口应尽量引用官方来源，例如全国人大法律法规数据库、中国政府网、人社部门、司法行政机关、法律援助中心、仲裁委等。
- 新增地方政策或本地办事指南时，必须标注省/市、来源链接、更新时间或最后核验日期。
- 对不确定、地区差异明显或裁审口径可能不同的内容，使用“需核验”“可能”“通常”“建议咨询当地机构”等谨慎措辞。

## 数据贡献要求

新增或更新 `data/` 下的公开数据时，请同步检查：

- 是否有 `sourceUrl`、`effectiveDate`、`lastReviewed`、`reviewer`、`verificationStatus` 等可追溯字段。
- 是否明确数据适用地区、档位、更新时间和可能的限制。
- 是否避免把启动版索引写成实时、权威或保证准确的数据。
- 是否同步更新相关文档，例如 [数据源与更新说明](docs/DATA_SOURCES.md)。

## Pull Request 检查清单

提交前请确认：

- [ ] 一个 PR 只解决一个主题。
- [ ] 示例、截图、文档中的案例已经完全脱敏。
- [ ] 新增法律或政策内容已尽量引用官方来源，并标注地区和更新时间。
- [ ] 修改金额计算、数据结构或导出逻辑时已补充测试。
- [ ] 已运行 `pnpm run validate` 和 `pnpm test`。
- [ ] 文书模板和提示词没有绝对化承诺，也没有暗示可以替代律师、法律援助机构或有权机关。

## 沟通方式

如果你不确定某个贡献是否合适，请优先开 issue 说明背景、数据来源、使用场景和风险边界。涉及具体个案的讨论请先脱敏，并避免公开传播完整证据材料。
