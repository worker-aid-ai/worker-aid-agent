# CHANGELOG

本项目采用接近 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.1.0/) 的格式记录重要变化。版本日期以仓库标签或明确发布记录为准；未打标签的版本会明确标注，避免误导使用者。

## 1.3.0 - 2026-06-30

### Added

- 新增 Agent 多轮问答状态结构，覆盖案情采集、证据补缺、金额估算、文书草稿和求助渠道。
- 新增欠薪、加班费、未签书面劳动合同、违法解除等场景的问题树，支持按轮次追问缺失事实，避免一次性要求用户提交大量敏感材料。
- 新增 CLI `agent <scenario_id> [answers.json]` 命令，用于输出缺失事实、风险提醒、人工复核提醒和模型发送前脱敏摘要。
- 新增本地 App 的 Agent 问答面板，支持从当前案情开始追问、提交本轮 JSON 回答、查看模型发送前摘要和生成材料草稿。
- 新增模型调用前摘要构造层，只展示必要且已脱敏的信息，并把被省略的敏感字段以脱敏形式呈现给用户确认。
- 新增“继续追问缺失事实”和“生成材料草稿”两类输出路径，优先使用 JSON/Markdown 便于人工复核。

### Changed

- README、Roadmap 和 CLI help 同步 Agent 问答能力；README 和本地 App 用户界面不暴露路线图版本号。
- 插件包 manifest 版本同步为 `1.3.0`。

### Notes

- Agent 问答仍然只用于材料整理和咨询前准备，不构成法律意见，不替代劳动者本人、法律援助机构、律师、工会、劳动监察或仲裁机构的判断。
- 所有模型输出和材料草稿都不得承诺投诉、仲裁、诉讼、调解、法律援助或协商结果，不得冒充律师或专业机构身份。
- 工具只整理真实已经存在的事实和材料，不生成、补写、篡改或诱导伪造证据。
- 调用外部模型前必须由用户确认即将发送的脱敏摘要；涉及事实、证据、金额、时效、管辖和地方政策的输出提交前仍需人工复核。

## 1.2.0 - 2026-06-30

### Added

- 新增 Windows 双击启动入口 `start-worker-aid.cmd`，面向普通用户降低本地 App 启动门槛。
- 新增本机草稿保存、恢复和清除能力，API key 不写入本机草稿，恢复后需重新填写并确认外部模型调用。
- 新增共享脱敏预检模块，提示完整手机号、身份证号、银行卡相关描述、住址和疑似原始聊天记录等敏感内容。
- 新增本地 Web 服务健康信息 `/health.json`，用于确认本地优先运行状态和安全提醒。
- 新增 Windows 启动器测试，覆盖双击入口和自动打开浏览器行为。

### Changed

- `start-worker-aid.ps1` 增加 Node.js、pnpm、端口占用检查，并在通过诊断后自动打开本地页面。
- 本地 App 首屏工作流补齐保存草稿、恢复草稿、清除草稿和更清晰的脱敏提醒。
- README、使用指南、本地 App 文档和 Roadmap 同步为 v1.2 本地 App 可用性完成状态。
- 插件包 manifest 版本同步为 `1.2.0`。

### Notes

- 本地 App 仍默认只在用户本机处理输入；调用外部模型前必须由用户确认，并只发送必要且已脱敏的摘要。
- 所有法律、金额、时效、证据、管辖和地方政策相关输出仍只是材料整理草稿，提交前必须人工复核。

- 新增法律援助机构、公益律师、工会志愿者协作指南，说明材料流转、隐私脱敏、复核清单和禁止事项。

### Changed

- Roadmap 标记 v1.0 文档、协议和协作指南基线已完成。
- 包版本更新为 `1.0.0`。

### Notes

- v1.0 的协议、Skill 结构和协作指南仍只用于劳动者自助材料整理与法律援助准备，不构成法律意见或结果承诺。
- 涉及时效、金额、管辖、证据证明力和地方政策的输出，仍必须由用户和具备相应职责的人类工作人员复核。

## 0.3.0 - 2026-06-16

### Added

- 新增本地 RAG 法条检索启动版，基于 `data/law-basis.json` 返回主题、条文提示、摘要和官方核验渠道。
- 新增本地化政策索引 `data/local-policy-index-cn.json`，记录省/市、更新时间、官方入口和检索关键词。
- 新增多模型评测集启动版 `data/model-evaluation-set.json`，覆盖免责声明、人工复核、不伪造证据、不承诺结果和隐私脱敏检查。
- 新增风险输出拦截器，检查冒充法律专业人士、承诺结果、伪造证据和缺少人工复核提醒等高风险内容。
- 新增社区案例脱敏工具与贡献流程文档，支持案情 JSON 的启动版自动脱敏和人工复核提醒。

### Changed

- CLI 新增 `law-search`、`local-policy`、`risk-check`、`eval-set` 和 `anonymize-case` 命令。
- `pnpm run validate` 扩展为检查 v0.3 数据、文档和安全模块入口。
- 包版本更新为 `0.3.0`。

### Notes

- v0.3 的法条和政策能力是本地索引启动版，不替代官方最新文本或专业人士复核。
- 自动脱敏和风险检查只做辅助筛查，公开或提交前仍需人工复核。

## 0.2.0 - 2026-06-15

### Added

- 新增本地静态 Web 表单界面，用于演示案情采集、金额估算、最低工资查询和材料草稿生成。
- 新增最低工资启动版数据与法律援助、劳动监察、人社服务、仲裁入口索引。
- 新增金额估算能力，覆盖工资欠付、未签书面劳动合同二倍工资差额、经济补偿、违法解除赔偿金和加班费等材料准备场景。
- 新增 Markdown、HTML 和 Word 可打开的 `.doc` HTML 文档导出能力。
- 新增案件时间线结构化输出、CLI 命令与 Web 可视化能力。
- 新增 Node 原生测试，覆盖金额计算、最低工资查询和文书导出等核心辅助能力。

### Changed

- CLI demo 扩展为包含最低工资查询示例。
- 文档补充数据源、Web UI、金额计算测试和 v0.2 能力说明。

### Notes

- 最低工资和服务入口数据为启动版索引，不保证实时准确。提交材料前应通过当地官方渠道二次核验。
- 金额估算只用于材料整理和沟通参考，不自动判断请求是否成立，也不构成法律意见。
