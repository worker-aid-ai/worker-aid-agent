# 社区案例脱敏贡献流程

社区案例只用于公益经验整理、功能评测和文档示例，不用于承诺个案结果。所有案例必须完全脱敏，并保留不确定性和人工复核提醒。

## 提交前检查

1. 使用本地脱敏命令生成初稿：

```bash
node src/worker-aid-cli.mjs anonymize-case path/to/case.json
```

2. 人工复核以下信息是否已经删除或替换：

- 劳动者真实姓名、手机号、身份证号、住址、银行卡号。
- 用人单位完整名称、统一社会信用代码、内部账号、客户信息。
- 未脱敏合同、工资条、银行流水、聊天记录截图、工牌、定位记录。
- 可反推出个人身份的时间、地点、岗位、金额组合。

3. 确认案例没有伪造、篡改或补造证据的内容。

4. 确认案例不包含“保证胜诉”“一定追回工资”等承诺结果表述。

## 推荐结构

```json
{
  "scenario": "wage_arrears",
  "region": "广东/深圳",
  "facts": {
    "startDate": "2025-03",
    "position": "技术岗位",
    "monthlySalaryRange": "8000-12000",
    "description": "劳动者A被某公司拖欠工资，已保存工资流水和催讨记录。"
  },
  "evidence": [
    "工资流水（已脱敏）",
    "催讨记录摘要（已脱敏）"
  ],
  "reviewNotes": [
    "公开前已人工复核隐私信息。",
    "本案例不构成法律意见，仅供材料整理参考。"
  ]
}
```

## 合并原则

- 只接受完全匿名、可公开讨论的材料摘要。
- 不接受原始证据文件、完整聊天记录、真实公司全称或可识别个人的信息。
- 不接受编造证据、诱导对抗或承诺结果的案例。
- 涉及地方政策时，必须标注省/市和更新时间，并尽量引用官方入口。
