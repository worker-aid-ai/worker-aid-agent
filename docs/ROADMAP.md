# Roadmap

本路线图遵循“独立本地 App 为主、专业协作者插件包为辅”的产品形态：普通劳动者优先使用本地向导整理材料，公益律师、法援志愿者、工会工作者和 AI power user 通过插件包复用同一套规则、数据和安全护栏。所有法律、金额、时效、证据和地方政策相关输出都必须保留不确定性说明和人工复核提醒。

## v1.1 本地 App 与插件包

- [x] 将普通用户入口升级为本地 App：案情向导、隐私提示、模型 API 配置和材料导出集中在首屏工作流中
- [x] 新增外部模型调用确认门，未勾选确认时不允许调用外部模型
- [x] 新增共享产品入口核心，用于规范本地会话、模型配置脱敏和人工复核提醒

- [x] 新增 Windows 本地启动脚本 `start-worker-aid.ps1`

## v1.2 本地 App 可用性与低门槛分发

目标：让普通非计算机用户更容易启动、理解和完成一次材料整理流程。

- [ ] 优化首屏向导，把“填写案情、检查隐私、生成材料、导出文件”做成更清晰的连续步骤
- [x] 增加本地会话保存/恢复能力，默认只保存在用户本机，并提供一键清除入口
- [x] 增加更完整的脱敏预检，覆盖完整手机号、身份证号、银行卡号、住址、公司内部账号和疑似原始聊天记录
- [x] 增加启动失败提示，针对 Node.js、pnpm、端口占用和浏览器打不开给出可执行修复建议
- [ ] 探索 Windows 免命令行启动包，优先保持本地运行和可审计，不引入不必要的安装器复杂度
- [ ] 验收标准：首次使用者能在不理解 CLI 的情况下启动页面、填写脱敏示例、生成摘要并下载 Markdown/HTML

## v1.3 Agent 化案情问答

目标：从静态表单升级为谨慎的多轮材料整理助手，但不替代法律判断。

- [ ] 设计本地 App 的多轮问答状态结构，覆盖案情采集、证据补缺、金额估算、文书草稿和求助渠道
- [ ] 将欠薪、加班费、未签合同、违法解除等场景拆成可复核的问题树，避免一次性要求用户提交大量敏感材料
- [ ] 接入模型调用前的摘要构造层，只发送必要且已脱敏的信息，并展示即将发送的内容供用户确认
- [ ] 增加模型输出风险检查，拦截承诺结果、伪造证据、冒充专业身份和缺少人工复核提醒的内容
- [ ] 支持“继续追问缺失事实”和“生成材料草稿”两类输出路径，输出优先使用 Markdown/JSON
- [ ] 验收标准：至少覆盖欠薪和违法解除两个完整问答流程，每一步都有缺失信息、风险提醒和人工复核提示

## v1.4 插件包安装与协作者工作流

目标：让专业协作者能在已有 AI 工具中稳定复用 Worker Aid 的 skills、agents、prompts 和安全规则。

- [ ] 定义插件包安装清单，明确 Codex、Claude、OpenClaw、Cursor 等运行时的目录映射和最小支持范围
- [ ] 增加安装/更新脚本或说明，支持预览将复制的文件，避免覆盖用户本地修改
- [ ] 为协作者补充典型工作流：案情复核、证据目录复核、文书草稿复核、风险输出检查和社区案例脱敏
- [ ] 增加插件包完整性校验，确认 manifest、skills、agents、prompts 和协议文档一致
- [ ] 验收标准：协作者能按文档把插件包接入至少一个 AI 运行时，并完成一条脱敏案情的复核流程

## v1.5 数据维护、评测与官方来源治理

目标：让法律依据、地方政策、最低工资和安全评测保持可更新、可追溯、可复核。

- [ ] 扩展数据更新脚本，记录来源、更新时间、适用地区、核验状态和维护人备注
- [ ] 增加数据质量检查，覆盖最低工资、服务入口、法条索引、本地政策索引和评测集
- [ ] 扩展模型评测集，覆盖更多高风险输出：诱导伪造证据、过度确定法律结论、泄露敏感信息和错误地方政策
- [ ] 建立公开案例贡献的审核清单，确保示例完全脱敏且不包含真实公司全称、联系方式或可识别个人信息
- [ ] 验收标准：新增或更新法律/政策数据时必须能通过自动校验，并在文档中说明官方核验入口

## v2.0 协作试点与桌面化评估

目标：在安全边界清晰的前提下，验证真实协作场景，并决定是否进一步桌面化。

- [ ] 与公益法律服务、工会志愿者或劳动者互助社区开展小范围脱敏试点，收集可用性和安全反馈
- [ ] 基于试点反馈改进普通用户文案、字段顺序、导出模板和协作者复核清单
- [ ] 评估桌面封装方案，比较 Tauri、Electron、PWA/快捷方式和继续本地 Web 的维护成本与隐私风险
- [ ] 明确桌面版是否需要自动更新、离线数据包、日志脱敏和本地模型支持
- [ ] 验收标准：形成试点报告和桌面化决策记录，不因分发便利牺牲隐私、安全和可审计性

## 持续护栏

- [ ] 所有法律相关输出必须包含不确定性说明和人工复核提醒
- [ ] 不生成、补写、篡改或诱导伪造证据
- [ ] 不承诺投诉、仲裁、诉讼、调解、法律援助或协商结果
- [ ] 不冒充律师、法律援助机构、工会、仲裁委、法院或行政机关
- [ ] 示例、文档和公开案例必须完全脱敏
- [ ] 新增法律或地方政策内容优先引用官方来源，并标明地区、更新时间和核验入口
